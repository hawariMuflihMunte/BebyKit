BebyKit

Introduction

BebyKit is an-all-new Front-end framework which may helps you tackle your Front-end development. Rapidly develop your project by using your imagination (free-design-mode) or by using framework's built-in Components.

Get started with BebyKit, mobile-first approach and responsive display.

Quick Start

Starter Template
Without Dongle Font
HTML 
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name=viewport content="width=device-width, initial-scale=1.0">

    <title>Project Name</title>

    <link rel=stylesheet href="assets/script/css/bebykit.min.css">
</head>
<body>
    
</body>
</html>
With Dongle Font
HTML 
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name=viewport content="width=device-width, initial-scale=1.0">

    <title>Project Name</title>

    <link rel=stylesheet href="assets/script/css/bebykit.min.css">

    <link rel=preconnect href="https://fonts.googleapis.com">
    <link rel=preconnect href="https://fonts.gstatic.com" crossorigin>
    <link rel=stylesheet href="https://fonts.googleapis.com/css2?family=Dongle:wght@300;400;700&display=swap">
</head>
<body>
    
</body>
</html>

If you choose and use BebyKit framework with Dongle font, you have to make an additional CSS code into your project.

CSS 
:root {
    --font-size-used: 15pt;
}

The font size above is optimized with the legacy code and will suits in your project.
Make sure to disable this, if you are not using Dongle font or if you are using other font (not default which is sans-serif) or Dongle font!

For the framework's CSS, you can download it from Google Drive (previewable) or instantly from here. Pick whatever suits to yours and have fun...

Google Drive Instant Download

If you have already downloaded the framework and its assets, use it in your project by placing its assets directly in your project.

Open-source

You can contribute to this project which is open for all designers & developers around the world.
If you want to contribute to this project, you can do that by doing these steps below:

Contribution Guideline

Instruction:

  • The very first, star the project repo.
  • Fork the project repo.
  • Create your own script.
  • Commit & Push your code.
  • Create Pull Request.
  • For developers whose involved convenient, all of the Pull Request will be discussed. If all the developers (or more than half) are agree with the merging script to the legacy code, then the Pull Request will be triggered after agreement.
  • Congratulations, you have just contributed to an open-source project.
